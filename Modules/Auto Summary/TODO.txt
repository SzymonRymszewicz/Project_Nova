AUTO SUMMARY MODULE ROADMAP (v1)

Goal
- Keep chat context under max token budget with minimal memory loss.
- Use 3 progressive summary phases.
- Preserve relative context order and traceability.
- Add user-controlled protected IAMs (Core Memories).

Key Principles
- Every IAM can go through Phase1 and Phase2 at most once.
- Unmodified IAMs (Phase0) are protected from Phase2 and Phase3.
- Protected IAMs (Core Memories) are protected from ALL summary phases.
- Relative ordering must remain stable after all transformations.
- Summaries keep source references (lineage) to original IAM ids.

--------------------------------------------------
1) Data Model and Tags
--------------------------------------------------

For each IAM, add/maintain metadata:
- iam_id: unique id
- phase: 0 | 1 | 2 | 3
- summarized_phase1: bool
- summarized_phase2: bool
- protected_core: bool (new)
- hidden_from_chat: bool
- source_iam_ids: list[str] (for grouped summaries)
- created_at, updated_at
- token_count

Rules:
- protected_core=true => never selected by Phase1/2/3.
- phase0 IAMs can only be selected by Phase1.
- phase1 IAMs can be selected by Phase2 and Phase3.
- phase2 IAMs can be selected by Phase3.

--------------------------------------------------
2) Trigger Strategy (with anti-thrashing)
--------------------------------------------------

Config values:
- Phase1_Trigger
- Phase2_Trigger
- Phase2_Goal
- Phase2_IAM_count
- Phase3_Trigger
- Phase3_Goal
- Phase3_IAM_count

Recommended controls:
- Trigger hysteresis: once triggered, do not retrigger until usage drops below (trigger - hysteresis_margin).
- Cooldown turns: optional minimum turns between same-phase runs.

Definitions:
- usage = total_context_tokens / max_context_tokens
- unmodified_usage = tokens of phase0 IAMs / max_context_tokens

--------------------------------------------------
3) Phase Logic
--------------------------------------------------

Phase1 (light compress)
- Condition: unmodified_usage > Phase1_Trigger.
- Select oldest eligible IAM:
	- phase == 0
	- protected_core == false
- Transform with deterministic light compression (remove filler/verbosity, preserve facts).
- Mark:
	- phase = 1
	- summarized_phase1 = true
	- hidden_from_chat = true
- Repeat per loop iteration as needed.

Phase2 (mid compress by groups)
- Condition:
	- usage > Phase2_Trigger
	- count(phase1, not protected_core) >= (Phase2_IAM_count + 1)
- Select oldest Phase1 IAMs (not protected) in groups of Phase2_IAM_count.
- Summarize each group into one new Phase2 IAM.
- Set lineage source_iam_ids from grouped IAMs.
- Preserve order by inserting new summary at first source position.
- Continue until usage <= Phase2_Goal OR no more valid Phase1 groups.

Phase3 (LLM compress reset)
- Condition:
	- usage > Phase3_Trigger
	- no valid Phase2 action possible (insufficient Phase1 IAMs)
- Eligible pool: oldest IAMs where phase in {1,2} and protected_core == false.
- Group up to Phase3_IAM_count and summarize using LLM + custom prompt + current API settings.
- Continue until usage <= Phase3_Goal OR no eligible IAMs.
- This acts as deep reset for old summarized context.

--------------------------------------------------
4) Core Memories UX (Protected IAM Tag)
--------------------------------------------------

Feature:
- On IAM hover (next to existing icons), show lock/unlock icon.
- Click toggles protected_core tag.

Behavior:
- unlocked icon = not protected
- locked icon = protected core memory
- toggle action writes immediately to IAM metadata
- show tooltip text: "Mark as Core Memory (excluded from auto summary)"

Backend/API:
- Add endpoint or action to toggle IAM protection:
	- input: bot_name, chat_id, iam_id, protected_core(bool)
	- output: success + updated IAM metadata

Summary engine constraint:
- Any IAM with protected_core=true is excluded in selectors for Phase1/2/3.

--------------------------------------------------
5) Chat UX During Phase3
--------------------------------------------------

When Phase3 starts:
- Disable send button and message input.
- Show thinking-style status:
	"Summarizing chat history. This might take a while..."
- Optionally show progress count: "Pass X / Y".

When Phase3 ends/fails:
- Re-enable input.
- Remove animation/status.
- If failed, show non-blocking error toast and keep last valid context.

--------------------------------------------------
6) Safety and Quality Guards
--------------------------------------------------

Before writing a summary IAM:
- Validate non-empty output.
- Keep key entities/numbers/dates where possible.
- Enforce max token target per phase.

After write:
- Recompute context usage.
- Keep lineage map (source_iam_ids) for audit/debug.

Failure policy:
- If summarization fails, skip that group and continue safely.
- Never delete source IAMs before successful replacement write.

--------------------------------------------------
7) Execution Order per Message (State Machine)
--------------------------------------------------

On each chat turn (or pre-generation hook):
1. Recalculate usage and phase-eligible pools.
2. Run Phase1 if needed.
3. Recalculate usage.
4. Run Phase2 loop while eligible and above Phase2_Goal.
5. Recalculate usage.
6. If still above threshold and Phase2 exhausted, run Phase3 loop.
7. Save final context stats.

Determinism:
- Oldest-first selection must be stable and deterministic.

--------------------------------------------------
8) Config File Plan
--------------------------------------------------

settings_auto_summary.txt should include:
- enabled=true/false
- phase1_trigger
- phase2_trigger
- phase2_goal
- phase2_iam_count
- phase3_trigger
- phase3_goal
- phase3_iam_count
- hysteresis_margin
- phase_cooldown_turns
- phase3_max_passes
- keep_lineage=true

--------------------------------------------------
9) Build Plan (Step-by-step)
--------------------------------------------------

Step 1: Metadata foundation
- Add IAM metadata schema.

Step 2: Selectors and eligibility
- Implement reusable selectors for phase candidates with protected_core exclusion.

Step 3: Phase1 engine
- Add deterministic light compression and hidden_from_chat behavior.

Step 4: Phase2 engine
- Add grouped summarization, replacement insertion, and goal loop.

Step 5: Phase3 engine
- Add LLM summarization loop, lock UI state, and progress events.

Step 6: Core memory toggle UX
- Add lock/unlock icon on IAM hover.
- Wire API action to set/unset protected_core.

Step 7: Trigger orchestration
- Implement state machine order and anti-thrashing controls.

Step 8: Validation and logging
- Add per-phase logs, counts, token deltas, and failure telemetry. Use existing debugging framework
using console.py and debug_manager.py

--------------------------------------------------
10) Acceptance Criteria
--------------------------------------------------

- Context remains under limits without breaking recent conversation quality.
- Protected (Core Memory) IAMs are never summarized.
- Phase0 IAMs are never touched by Phase2/3.
- Phase1/2 one-time constraints are respected.
- Relative ordering and source lineage are preserved.
- Phase3 locks input, shows status, and always unlocks on completion/failure.
